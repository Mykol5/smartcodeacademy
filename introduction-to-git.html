Introduction to Git & GitHub

Lecture: Introduction to Git and GitHub

Git is a distributed version control system that helps you track changes in your code and collaborate with others. GitHub, on the other hand, is a web-based platform that provides hosting for software development and a centralized location for version-controlled code. In this lecture, you'll learn the basics of Git and GitHub.

1. Getting Started with Git

If you haven't installed Git on your machine, visit the Official Git website and follow the instructions for your operating system.

  1. Visit the following link for a pictorial guide on how to install Git on Windows: How to Install Git on Windows
  2. Open a terminal in your development environment.
  3. Create a new folder for your project: mkdir git-intro
  4. Navigate into the project folder: cd git-intro
  5. Initialize a new Git repository: git init

2. Basic Git Commands

Learn some fundamental Git commands. For more details, you can refer to the GitHub documentation:

  1. Create a new file: touch index.html
  2. Add the file to the staging area: git add index.html
  3. Commit the changes: git commit -m "Initial commit"
  4. View the commit history: git log

3. Creating a GitHub Repository

Go to the GitHub website (https://github.com/) and create a new account if you don't have one. Follow these steps to create a new repository:

  1. Click on the "+" sign in the top right corner and select "New repository."
  2. Give your repository a name (e.g., git-intro) and add a description.
  3. Initialize this repository with a README if prompted.
  4. Click "Create repository."

4. Connecting Git with GitHub

Follow these steps to connect your local Git repository with a GitHub repository:

  1. On the GitHub website (https://github.com/), create a new repository without initializing it with a README.
  2. Copy the repository URL from GitHub.
  3. Back in your terminal, navigate to your project folder.
  4. Run the following commands:
            
            # Initialize a new Git repository in your project folder
            git init
            
            # Stage the file for commit
            git add .
            
            # Commit the changes
            git commit -m "Initial commit"
            
            # Set the remote repository URL (replace [repository-url] with the copied URL from your GitHub)
            git remote add origin [repository-url]
            
            # Set the default branch name to 'main'
            git branch -M main
            
            # Push your local commits to GitHub
            git push -u origin main

Now your local Git repository is connected to the GitHub repository, and you've pushed your initial commit to GitHub. You can continue making changes to your code, staging, committing, and pushing those changes to GitHub using the familiar Git commands.
